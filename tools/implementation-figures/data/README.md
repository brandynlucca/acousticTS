This directory should now hold only small committed numeric inputs that are
directly needed to build package-facing implementation figures.

What belongs here:

- compact tables read directly by vignette-facing figure builders
- small committed compare tables that are part of the public package docs

What has moved out:

- canonical `data/bem/`
- canonical `data/fem/`
- generated meshes
- large external-validation compare suites
- the old heavy `psms` gas-BEM validation outputs
- most of the legacy `tmm` external-validation artifact surface
- raw acoustic/reference/timing source tables that were only used to rebuild
  external compare suites

Those heavier assets now live in:

- [acousticTSValidation](C:/Users/Brandyn/Desktop/acousticTSValidation)

The remaining `data/tmm/` surface is now limited to the small pure-R outputs
that are still generated by the package-side `tmm` builders.

The remaining root-level compare CSVs are kept here intentionally because the
surviving package-side builders and public vignettes still read them directly.

There is no longer any package-side `data/fcms/` or `data/psms/` benchmark
validation surface. Those generated validation products now live only in the
external validation repo.
