This directory stores committed CSV inputs and comparison summaries consumed by
the implementation vignettes.

These files are intentionally versioned with the vignette assets so that
implementation pages do not depend on ephemeral `scratch/` outputs during site
builds.

If a static implementation figure or table needs committed numeric inputs, they
should live here rather than under `scratch/`.

The `tmm/` subdirectory stores the committed comparison inputs and summaries
used by the heavier `TMM` implementation validation figures.
